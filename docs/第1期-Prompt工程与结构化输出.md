## 技术要点1：System Prompt设计

客服的Prompt不是随便写两句"你是一个客服"就完事了，企业级的Prompt需要包含：

- 角色设定（你是谁、语气风格）
- 能力边界（能做什么、不能做什么）
- 回复规范（先安抚再解决、主动提供选项）
- 安全约束（不编造信息、不越权承诺）

这四块缺一不可，面试问到Prompt Engineering能答出这几个维度就比大多数人强了

---

## 技术要点2：Structured Output

大模型默认返回纯文本，但实际业务中你需要结构化数据

比如客服场景，除了回复内容你还需要知道：用户意图是什么（查订单？退货？投诉？）、置信度多高、要不要转人工

OpenAI SDK 的 `response_format` 配合 Pydantic BaseModel，让模型直接输出严格JSON，不用自己写正则解析，返回就是Python对象直接 `.intent` `.reply` 点出来用

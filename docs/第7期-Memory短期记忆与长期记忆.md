## 短期记忆和对话历史有什么区别

很多人觉得把聊天记录存下来就是"记忆"了，其实不是一回事

对话历史是原始消息流，token 越聊越多，要靠 summary 压缩才不会爆上下文。但压缩是有损的，"用户是钻石会员"这种关键信息可能压着压着就丢了

短期记忆是另一层抽象：每轮对话结束后，用 LLM 从最近的消息里提取结构化事实——身份、偏好、情绪、关注的商品。这些事实独立于对话历史存在，不会被 summary 压缩吃掉，每轮都作为 system message 注入 prompt

你可以把对话历史理解成"流水账"，短期记忆是"笔记"。流水账会被压缩甚至丢弃，但笔记一直在



## 长期记忆的写入时机

长期记忆要跨会话持久化，关键设计点是：什么时候写入？

一种做法是每轮都写，但这样提取频率太高，成本大，而且对话还没结束就下结论容易提取到半成品信息（用户说"我想退货"但最后没退）

我们选的是会话结束时一次性提取。整段对话都结束了，再让 LLM 回顾全程，提取真正值得记住的用户画像：身份、偏好、行为模式、历史问题。这时候信息最完整，提取质量最高

下次用户来的时候，加载他的长期记忆注入 prompt，小夕开口就能说"上次您咨询的那款手机，后来买了吗？"——这就是 Memory 带来的体验差异

面试被问到 Memory 设计，能讲清楚"会话内实时提取 vs 会话结束批量沉淀"这个分层，比只说"我用了 mem0"要有说服力得多

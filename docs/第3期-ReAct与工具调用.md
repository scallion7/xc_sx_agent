## 什么是 ReAct

ReAct = Reasoning + Acting，大模型领域最经典的 Agent 范式，没有之一

核心思想很简单：模型不再一次性给答案，而是进入一个循环——先想（我需要什么信息），再做（调工具拿数据），看结果，再想，再做……直到信息够了，给出最终回复

举个真实的例子，用户说"帮我查一下订单 ORD-001 的物流到哪了"，Agent 的过程是这样的：

💭 思考：用户要查物流，我需要调用物流查询工具
🔧 调用：query_logistics(order_id="ORD-001")
📋 结果：顺丰 SF123456，1月18号到了上海浦东，正在派送
💭 思考：拿到数据了，可以回复了
🤖 回复：您的快递正在上海浦东派送中…

这个"想-做-看"的循环就是 ReAct。面试被问到 Agent 架构，这个必须能讲清楚



## 工具调用怎么接

OpenAI 原生支持 Function Calling，你定义好工具的 schema（名字、参数、描述），传给模型，模型会自己判断什么时候需要调工具、调哪个、传什么参数

这期实现了四个电商场景的工具：查订单、查商品、查物流、申请退款

有个设计上的取舍值得聊：ReAct 循环中间产生的工具调用和返回结果，要不要存进对话历史？

我一开始想的是不存，只保留最终回复，这样历史干净。但马上发现问题——用户第一轮查了订单拿到了快递单号，第二轮问"物流到哪了"，如果历史里没有上一轮的工具返回，模型根本不知道快递单号是多少，得重新调一次查订单

所以最后选择了全量保存。中间的推理、工具调用、工具结果全部留在历史里。历史增长快了没关系，上期做的 Summary 压缩机制正好派上用场
